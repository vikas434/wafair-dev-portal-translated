Queries

fulfillmentOrderDetails

Description

Gibt Bestellinformationen einschließlich Status für die Multi-Channel-Fulfillment-Bestellung eines Lieferanten zurück.

Response

Returns a FulfillmentOrderDetails

Arguments
Name Description
orderDetailsInput - FulfillmentOrderDetailsInput!

Example

Query
query fulfillmentOrderDetails($orderDetailsInput: FulfillmentOrderDetailsInput!) {
  fulfillmentOrderDetails(orderDetailsInput: $orderDetailsInput) {
    fulfillmentOrder {
      ...FulfillmentOrderFragment
    }
    fulfillmentOrderErrors {
      ...FulfillmentOrderErrorFragment
    }
  }
}
Variables
{"orderDetailsInput": FulfillmentOrderDetailsInput}
Response
{
  "data": {
    "fulfillmentOrderDetails": {
      "fulfillmentOrder": FulfillmentOrder,
      "fulfillmentOrderErrors": [FulfillmentOrderError]
    }
  }
}

FulfillmentOrder

Description

Die allgemeinen Informationen zu einer Multi-Channel-Fulfillment-Bestellung, einschließlich ihres Status.

Fields
Field Name Description
requestId - String! Die Kennung, die von einer erfolgreichen Mutation createFulfillmentOrder zurückgegeben wurde, zur Identifizierung dieser Bestellung.
status - String! Der Status der Fulfillment-Bestellung.

PENDING: Bestellanforderung wurde erfolgreich empfangen (sowohl syntaktisch als auch semantisch überprüft).
NEW: Die Bestellung wurde erfolgreich bestätigt und im CastleGate-Fulfillmentsystem von Wayfair erstellt, aber es fehlen möglicherweise noch einige Fulfillment-Validierungsprüfungen.
ALLOCATED: Die Fulfillment-Bestellung wurde validiert. Alle Artikel der Bestellung wurden aus dem Lagerbestand zugewiesen. Wenn ein Artikel noch nicht zugewiesen wurde, bleibt der Bestellstatus auf NEW, bis alle Artikel zugewiesen sind.
PARTIALLY_SHIPPED: Einige Artikel der Bestellung wurden versendet.
SHIPPED: Alle Artikel der Bestellung wurden versendet.
REJECTED: Die Bestellung wurde aufgrund von Fehlern abgelehnt. Fehlerdetails finden Sie im Typ FulfillmentOrderDetails über die Abfragen fulfillmentOrderDetails oder fulfillmentOrderDetailsList.
ON_HOLD: Die Bestellung wurde zurückgestellt.
CANCELLED: Die Bestellung wurde storniert.
orderDate - DateTime Das Datum und die Uhrzeit, zu der die Bestellanforderung von Wayfair akzeptiert wurde.
customerOrderNumber - String Die Bestellnummer, die dem Kunden angezeigt wird, eingereicht mit der Mutation createFulfillmentOrder.
retailer - FulfillmentOrderRetailer Informationen zum Einzelhändler dieser Fulfillment-Bestellung.
shippingAddress - FulfillmentOrderShippingAddress Die Versandadresse, die mit der Mutation createFulfillmentOrder eingereicht wurde.
fulfillmentOrderItems - [FulfillmentOrderItem!] Liste der Artikeldetails, die in der Fulfillment-Bestellung enthalten sind.
Example
{
  "requestId": "xyz789",
  "status": "abc123",
  "orderDate": "2007-12-03T10:15:30Z",
  "customerOrderNumber": "xyz789",
  "retailer": FulfillmentOrderRetailer,
  "shippingAddress": FulfillmentOrderShippingAddress,
  "fulfillmentOrderItems": [FulfillmentOrderItem]
}

 Die Bestellung wurde erfolgreich bestätigt und im CastleGate-Fulfillmentsystem von Wayfair erstellt, aber es fehlen möglicherweise noch einige Fulfillment-Validierungsprüfungen.

ALLOCATED

 Die Fulfillment-Bestellung wurde validiert. Alle Artikel der Bestellung wurden aus dem Lagerbestand zugewiesen. Wenn ein Artikel noch nicht zugewiesen wurde, bleibt der Bestellstatus auf NEW, bis alle Artikel zugewiesen sind.

PARTIALLY_SHIPPED

 Einige Artikel der Bestellung wurden versendet.

SHIPPED

 Alle Artikel der Bestellung wurden versendet.

REJECTED

 Die Bestellung wurde aufgrund von Fehlern abgelehnt. Fehlerdetails finden Sie im Typ FulfillmentOrderDetails über die Abfragen fulfillmentOrderDetails oder fulfillmentOrderDetailsList.

ON_HOLD

 Die Bestellung wurde zurückgestellt.

CANCELLED

 Die Bestellung wurde storniert.
Example
"PENDING"
