Queries

fulfillmentOrderDetails

Description

返回有关供应商的多渠道履行订单的信息，包括订单状态。

Response

Returns a FulfillmentOrderDetails

Arguments
Name Description
orderDetailsInput - FulfillmentOrderDetailsInput!

Example

Query
query fulfillmentOrderDetails($orderDetailsInput: FulfillmentOrderDetailsInput!) {
  fulfillmentOrderDetails(orderDetailsInput: $orderDetailsInput) {
    fulfillmentOrder {
      ...FulfillmentOrderFragment
    }
    fulfillmentOrderErrors {
      ...FulfillmentOrderErrorFragment
    }
  }
}
Variables
{"orderDetailsInput": FulfillmentOrderDetailsInput}
Response
{
  "data": {
    "fulfillmentOrderDetails": {
      "fulfillmentOrder": FulfillmentOrder,
      "fulfillmentOrderErrors": [FulfillmentOrderError]
    }
  }
}

FulfillmentOrder

Description

关于多渠道履行订单的常规信息，包括其状态。

Fields
Field Name Description
requestId - String! 从成功的 createFulfillmentOrder 操作返回的标识符，用于标识此订单。
status - String! 履行订单的状态。

PENDING: 订单请求已成功接收（通过语法和语义验证）。
NEW: 订单已成功确认并在 Wayfair 的 CastleGate 履行系统中创建，但可能仍需通过某些履行验证检查。
ALLOCATED: 履行订单已通过验证。订单的所有物品均已从库存中分配。如果订单中有一个物品尚未分配，则订单状态将保持为 NEW，直到所有物品都已分配。
PARTIALLY_SHIPPED: 订单中的部分物品已发货。
SHIPPED: 订单中的所有物品已发货。
REJECTED: 订单由于某些错误被拒绝。错误详细信息可以通过 fulfillmentOrderDetails 或 fulfillmentOrderDetailsList 查询查看 FulfillmentOrderDetails 类型。
ON_HOLD: 订单已被暂挂。
CANCELLED: 订单已被取消。
orderDate - DateTime Wayfair 接受订单请求的日期和时间。
customerOrderNumber - String 客户订单号，由 createFulfillmentOrder 操作提交。
retailer - FulfillmentOrderRetailer 此履行订单的零售商信息。
shippingAddress - FulfillmentOrderShippingAddress 由 createFulfillmentOrder 操作提交的送货地址。
fulfillmentOrderItems - [FulfillmentOrderItem!] 履行订单中包含的物品详细信息列表。
Example
{
  "requestId": "xyz789",
  "status": "xyz789",
  "orderDate": "2007-12-03T10:15:30Z",
  "customerOrderNumber": "abc123",
  "retailer": FulfillmentOrderRetailer,
  "shippingAddress": FulfillmentOrderShippingAddress,
  "fulfillmentOrderItems": [FulfillmentOrderItem]
}
